# ANDP Agent Pack — Build & Deploy Skill

## §1 Overview

The ANDP agent pack consists of:

| File | How produced | Edit manually? |
|------|-------------|---------------|
| `skills/user/ai-native-data-product/ai-native-data-product.md` | Assembled by `build.py` from source documents | **Never** — re-run build.py |
| `skills/user/ai-native-data-product/skill.json` | Generated by `build.py` from embedded template | Only to change skill metadata |
| `agent_packs/andp/manifest.json` | Generated by `build.py` from embedded template | Only via `build_manifest()` in build.py |
| `agent_packs/andp/andp.agentpack` | Packaged by `build.py` | Never |
| `agent_packs/andp/base_content.md` | LLM-authored once | Only when ANDP architecture changes |
| `agent_packs/andp/compressed_modules/{module}.md` | LLM-authored (compressed mode only) | Only when source docs change and compressed mode is in use |

**Source of truth for skill content:** `../ai-native-data-products/design-standards/*.md` (6 module design standard documents).

**Conversion prompt:** `../ai-native-data-products/prompts/Skill_Conversion_Prompt.md` — governs all content quality rules.

## §2 Consistency Checks (run when source documents change)

Before re-running build.py after any source document update, verify these rules across the changed document(s). Report every violation to the user and wait for correction before proceeding.

| Check | Rule | Failure pattern |
|-------|------|----------------|
| Boolean type | `BYTEINT NOT NULL DEFAULT 0\|1` | `CHAR(1)`, `VARCHAR`, `INTEGER` |
| Boolean prefix | Column name starts with `is_` | `active`, `deleted`, `current` (no prefix) |
| Boolean filter | `= 1` or `= 0` | `= 'Y'`, `= 'N'`, `= TRUE` |
| Temporal PI | `PRIMARY INDEX` (non-unique) | `UNIQUE PRIMARY INDEX` on temporal table |
| Timestamp type | `TIMESTAMP(6) WITH TIME ZONE` | Bare `TIMESTAMP`, `TIMESTAMP(0)` |
| Domain surrogate | `BIGINT NOT NULL` for FK-target `_H` entities | `GENERATED ALWAYS AS IDENTITY` on `_H` |
| Other module surrogates | `GENERATED ALWAYS AS IDENTITY` | `BIGINT NOT NULL` on management tables |

## §3 Validate Conversion Prompt for Drift

Run this check when the user asks "is the skill up to date with the conversion prompt?" or when `Skill_Conversion_Prompt.md` is known to have changed.

1. Read `../ai-native-data-products/prompts/Skill_Conversion_Prompt.md`
2. Compare each rule/section against the current ANDP SKILL.md and `base_content.md`
3. For each rule, classify as:
   - **COVERED** — already enforced
   - **GAP** — relevant and applicable but missing
   - **N/A (architecture)** — specific to the multi-file `SKILL.md + modules/` format; ANDP uses Uderia param blocks
   - **N/A (verbatim)** — only applies to compressed content; not relevant when verbatim mode is in use
4. Report all GAPs with a one-line recommendation
5. Present findings to the user before making any changes

## §5 Normal Operation

### Step 1 — Choose content mode

Ask the user:

> "Do you want **verbatim** (exact source documents, ~6,500 lines, no omissions) or
> **compressed** (agent-optimised, ~2,800 lines, follows Skill_Conversion_Prompt.md
> compression rules) content?"

**Verbatim** is the default. **Compressed** requires that `agent_packs/andp/compressed_modules/{module}.md` files exist (LLM-generated; see §6 compressed workflow).

### Step 2 — Run build.py

**Verbatim (local server):**
```bash
python agent_packs/andp/build.py --import
```

**Verbatim (remote server):**
```bash
python agent_packs/andp/build.py --import --url http://192.168.0.46:5050 --password '><your-password>'
```

**Compressed (local server):**
```bash
python agent_packs/andp/build.py --compressed --import
```

**Compressed (remote server):**
```bash
python agent_packs/andp/build.py --compressed --import --url http://192.168.0.46:5050 --password '><your-password>'
```

**Build only (no import):**
```bash
python agent_packs/andp/build.py [--compressed]
```

### Compressed mode — cache check and regeneration

Compressed module files in `agent_packs/andp/compressed_modules/` are cached artefacts.
Only regenerate a module when its source document is newer than its cached file.

**Step 1 — Cache check (always run first)**

Check whether the cache directory exists:
```bash
ls agent_packs/andp/compressed_modules/ 2>/dev/null || echo "(directory does not exist)"
```

If the directory is missing or any module file is missing → that module is automatically stale (no stat needed).

For modules where the compressed file exists, compare source doc mtime vs compressed file mtime:

```bash
# Source doc mtime (git)
git -C ../ai-native-data-products log --format="%ai" -1 -- design-standards/Memory_Module_Design_Standard.md
# ... repeat for each module

# Compressed file mtime (filesystem — only if file exists)
stat -f "%Sm" -t "%Y-%m-%d %H:%M:%S" agent_packs/andp/compressed_modules/memory.md
# ... repeat for each module
```

Produce a status table and present to the user before doing any LLM work:

| Module | Source last modified | Compressed file | Status |
|--------|--------------------|--------------------|--------|
| memory | 2026-04-15 04:39 | 2026-04-20 11:00 | ✓ Cache valid |
| semantic | 2026-04-21 12:11 | 2026-04-19 09:00 | ✗ Stale — regenerate |
| domain | 2026-04-10 06:10 | (missing) | ✗ Missing — generate |
| ... | | | |

If all modules are cache-valid → skip regeneration entirely, run `build.py --compressed --import` directly.

**Step 2 — Regenerate stale modules only**

Ensure the cache directory exists before writing any compressed file:
```bash
mkdir -p agent_packs/andp/compressed_modules
```

For each stale module (confirmed by user):

1. Run §2 consistency checks on the source document
2. Read the source design standard and apply compression rules from `Skill_Conversion_Prompt.md` Step 3–4:
   - Replace prose explanations → decision tables
   - Replace specific examples → parameterised templates (`{ProductName}`, `{entity}`)
   - Consolidate repeated column sets (define temporal columns once, reference elsewhere)
   - **Preserve exactly** (do not paraphrase): validated SQL marked `TESTED ✅`, `v_relationship_paths` recursive CTE, `TD_VectorDistance()` call patterns, `lineage_graph` view CASTs, Documentation Sub-Module table schemas
   - Target: 300–400 lines per module file; hard limit: 500 lines
3. Write output to `agent_packs/andp/compressed_modules/{module}.md`

**Step 3 — Build and deploy**

```bash
python agent_packs/andp/build.py --compressed --import [--url ... --password ...]
```

**Force regeneration** — bypass cache for all modules:
Ask the user to confirm, then regenerate all 6 regardless of timestamps.

Compressed module files are committed to the repo.

## §7 When to Update base_content.md

`base_content.md` is the only LLM-authored piece that ships in both verbatim and compressed modes. It is loaded by the coordinator and by all 6 slave profiles as their base context.

**Update `base_content.md` only when:**
- A new module is added to the architecture
- A slave profile tag is renamed
- The routing logic changes (a query pattern should go to a different slave)
- A universal DDL convention changes
- A documentation capture protocol rule changes

**Do NOT update `base_content.md` when:**
- Module design standard documents are updated — those changes flow through automatically via build.py (verbatim mode) or via compressed module regeneration (compressed mode)

After editing `base_content.md`, run build.py to regenerate and reimport.

## §8 Profile Architecture

| Tag | profile_type | role | Skill context loaded |
|-----|-------------|------|---------------------|
| `ANDP` | `genie` | coordinator | base content only (routing table + architecture overview) |
| `ANDP-DOMAIN` | `llm_only` | expert | base + full `Domain_Module_Design_Standard.md` |
| `ANDP-SEMANTIC` | `llm_only` | expert | base + full `Semantic_Module_Design_Standard.md` |
| `ANDP-SEARCH` | `llm_only` | expert | base + full `Search_Module_Design_Standard.md` |
| `ANDP-PREDICTION` | `llm_only` | expert | base + full `Prediction_Module_Design_Standard.md` |
| `ANDP-OBSERVABILITY` | `llm_only` | expert | base + full `Observability_Module_Design_Standard.md` |
| `ANDP-MEMORY` | `llm_only` | expert | base + full `Memory_Module_Design_Standard.md` |

**Coordinator genieConfig:** `temperature: 0.3`, `queryTimeout: 600`, `maxIterations: 12`

**skillsConfig format:** Coordinator uses `"params": {}` (empty — base only). Each slave uses `"params": {"param": "<module>"}`.

**Genie pass-through:** Single-expert queries with no conversation history skip the coordinator synthesis LLM call — the slave's answer is returned directly (zero overhead).

## §10 Quality Checklist

**For base_content.md changes:**
- [ ] Architecture table covers all 6 modules with correct database pattern
- [ ] Deployment order correct: Phase 1 = Memory+Semantic, Phase 2 = Domain+Observability, Phase 3 = Search+Prediction
- [ ] Coordinator routing table covers all 6 modules — each query pattern maps to exactly one slave (or "both" for cross-module)
- [ ] No DDL templates or SQL code blocks in base_content.md (coordinator gets routing intelligence, not DDL)
- [ ] Documentation capture protocol includes all record types: MOD-, DD-, CL-, BG-, QC-, IN-
- [ ] After editing: run build.py and verify `profiles_created: 7`

**For compressed module files (compressed mode only):**
- [ ] All 6 compressed module files present under `agent_packs/andp/compressed_modules/`
- [ ] Each module file ≤ 500 lines
- [ ] Does NOT repeat universal conventions from base_content.md
- [ ] All tables from the source design standard are present
- [ ] DDL uses `{ProductName}` and `{entity}` placeholders throughout
- [ ] No `CHAR(1)` boolean columns; no `= 'Y'` or `= 'N'` filter values
- [ ] Validated SQL (marked `TESTED ✅` in source) preserved exactly
- [ ] Semantic registration INSERTs present
- [ ] Memory documentation INSERTs use `{ProductName}_Memory.` prefix
- [ ] Expire-current → insert-new DML pattern present for temporal tables
- [ ] Staleness detection query present
- [ ] Agent discovery query sequence present

**Packaging check (always):**
- [ ] `skill.json` and `ai-native-data-product.md` inside `.skill` zip at root level (no subdirectory)
- [ ] `.agentpack` contains `manifest.json` + `skills/ai-native-data-product.skill`
- [ ] Coordinator `skillsConfig` has `"params": {}` (not `"param": "..."`)
- [ ] All 6 slave profiles have `"params": {"param": "<module>"}` in `skillsConfig`
